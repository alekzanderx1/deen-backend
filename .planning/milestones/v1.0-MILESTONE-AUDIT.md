## Tech Debt (6 items, 0 blockers)

### Phase 04 — Assembly and Integration (4 items)

1. **INTG-03 UX accuracy** — Pre-canned SSE status events always emit `fiqh_refine` regardless of whether the sub-graph actually ran a refinement iteration. `FiqhState.status_events` is populated in the sub-graph but `_call_fiqh_subgraph_node` does not return it in the ChatState delta, so `pipeline_langgraph.py` cannot read live events. **Severity:** Low — UX inaccuracy only, no crash.

2. **AGEN-08 maintenance risk** — Streaming path in `pipeline_langgraph.py` duplicates `generate_answer()` logic inline (evidence formatting, disclaimer appending, citation extraction). If `generator.py` logic changes, the streaming path silently diverges. **Severity:** Low — current behavior is correct; risk is future divergence.

3. **Stale import** — `agents/core/chat_agent.py` line 20 imports `EARLY_EXIT_FIQH` (now unused after Phase 4 replaced the node with LLM-generated rejection). **Severity:** Cosmetic.

4. **Latent routing trap** — `_fiqh_classification_node` writes `is_fiqh=True` for VALID_* queries. `_should_continue` routes any state with `is_fiqh=True` to `"exit"`. Current graph topology prevents VALID_* queries from reaching `_should_continue`, but future graph changes could trigger incorrect routing. **Severity:** Low — not a current defect.

### Phase 03 — FAIR-RAG Core Modules (1 item)

5. **Dead code** — `modules/fiqh/fair_rag.py:run_fair_rag()` is never called by any live code path (superseded by fiqh_graph.py per D-03). Has 9 passing unit tests. **Severity:** Low — safe to retain or remove.

### Phase 01 — Data Foundation (1 item)

6. **Cosmetic comment** — `scripts/ingest_fiqh.py` line 236 has stale `# Ingestion stub` comment. **Severity:** Cosmetic.
