## Answer Streaming

| Option | Description | Selected |
|--------|-------------|----------|
| Add generate_answer_stream() to generator.py | New streaming variant using chain.stream() | |
| Stream at pipeline layer, not in sub-graph | Sub-graph calls generate_answer() synchronously; pipeline re-runs LLM for streaming | |
| Single-chunk emit | Full answer string as one response_chunk event | |

**User's clarification:** Rejected all original options. User wants no unnecessary latency — the final response should be generated and streamed **once only**.

**Resolved approach:**
- Sub-graph handles decompose → retrieve → filter → assess → refine loop, then exits
- Sub-graph stores `fiqh_filtered_docs` + `fiqh_sea_result` in state
- Main graph has a new `generate_fiqh_response` node that performs the single streaming generation using the fiqh-specific system prompt from `generator.py`
- User explicitly noted the prompt must be fiqh-specific — generic `generator_prompt_template` would not work for fiqh answers
