From db/config.py — Pydantic Settings reads these env vars (already in place, do not modify):
```python
class Settings(BaseSettings):
    DB_HOST: str = Field(..., validation_alias=AliasChoices("DB_HOST", "POSTGRES_HOST", "PGHOST"))
    DB_PORT: int = Field(5432, validation_alias=AliasChoices("DB_PORT", "POSTGRES_PORT", "PGPORT"))
    DB_NAME: str = Field(..., validation_alias=AliasChoices("DB_NAME", "POSTGRES_DB", "PGDATABASE"))
    DB_USER: str = Field(..., validation_alias=AliasChoices("DB_USER", "POSTGRES_USER", "PGUSER"))
    DB_PASSWORD: str = Field(..., validation_alias=AliasChoices("DB_PASSWORD", "POSTGRES_PASSWORD", "PGPASSWORD"))
```

From db/session.py — sync engine built at import time with `connect_args={"sslmode": "require"}`. Misconfigured `.env` -> import error at server boot.

Direct connection DSN pattern (from CONTEXT canonical_refs):
- Host: `db.[project-ref].supabase.co`
- Port: `5432`
- Database: `postgres`
- User: `postgres`
- Password: from Supabase Dashboard -> Project Settings -> Database
- ASYNC DSN: `postgresql+asyncpg://postgres:[PWD]@db.[project-ref].supabase.co:5432/postgres`

NOT the pooler host `aws-0-[region].pooler.supabase.com:6543` — that is the transaction pooler and is incompatible with asyncpg per v1.1 decisions.
