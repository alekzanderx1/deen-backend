<objective>
Fix the Supabase Admin API 401 in `DELETE /account/me`. Currently the call to delete a user from Supabase Auth sends only an `Authorization: Bearer …` header — Supabase's Kong gateway rejects this at the edge with HTTP 401 ("No API key found in request") because Kong requires an `apikey` header for tenant routing before the request ever reaches GoTrue.

Purpose: Without this fix, deleting an account leaves an orphaned Supabase auth row, so the user cannot re-sign-up with the same email. The endpoint still returns 204 (the Supabase failure is logged-but-swallowed per D-05), making the bug silent at the HTTP level but user-visible in practice.

Output: A single-line addition to the headers dict in `api/account.py` (Step 3, lines 83–86) so that both `apikey` and `Authorization` carry the service role key value. No other changes.
</objective>
