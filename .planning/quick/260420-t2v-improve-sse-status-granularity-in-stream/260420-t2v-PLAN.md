<objective>
Increase the frequency and granularity of SSE `status` events emitted by `/chat/stream/agentic` so the frontend can show live, human-readable progress while the agent works. Today, long-running stages (fiqh classification, fiqh sub-graph iterations, tool execution, generation preamble) complete silently between node boundaries, and the frontend is stuck on messages like "Classifying fiqh query" for several seconds.

Purpose: Better perceived responsiveness and debuggability for the end user. No new contract — only more `status` events interleaved with the existing stream.

Output: Updated `core/pipeline_langgraph.py`, `agents/core/chat_agent.py`, `agents/state/chat_state.py`, `agents/fiqh/fiqh_graph.py`, and a small extension to `tests/test_agentic_streaming_sse.py`. All other SSE event shapes and downstream consumers (`chat_persistence_service.wrap_streaming_response_for_persistence`) remain unchanged.
</objective>
